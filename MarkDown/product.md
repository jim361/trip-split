# Trip Split Product Draft

## 1. 제품 한 줄 설명

국내 여행을 함께 계획하고, 장소와 일정에 연결된 실제 지출을 실시간으로 정산하는 공동 여행 플래너.

## 2. 만들고 싶은 문제 해결

여행을 준비할 때 장소는 네이버 지도에 저장하고, 일정은 메모장이나 엑셀에 정리하고, 비용 정산은 카카오톡이나 별도 정산 앱으로 처리하는 경우가 많다. 이 과정에서 정보가 흩어지고, 실제 여행 중에는 누가 어디서 얼마를 냈는지 놓치기 쉽다.

Trip Split은 여행 장소, 시간표, 지도 동선, 지출 정산을 하나의 Firebase 기반 여행 세션으로 묶어 여행 전후의 번거로움을 줄이는 것을 목표로 한다.

## 3. 초기 타겟

- 국내 1박 2일 또는 2박 3일 단체 여행
- 친구, 동호회, 가족, 소규모 팀 여행
- 차량 이동이 포함되고, 여러 명이 번갈아 결제하는 여행
- 여행 전에는 장소와 일정을 함께 정리하고, 여행 후에는 정산을 깔끔하게 끝내고 싶은 사용자

## 4. MVP 방향

초기 MVP는 국내 여행에 집중한 모바일 우선 웹/PWA다. PC에서도 같은 내비게이션과 핵심 흐름을 유지하되, 넓은 화면에서는 정보 영역을 확장 배치한다.

- Firebase 기반 여행 세션 생성
- 사용자-facing 로그인 없는 시작 경험
- 내부 Firebase 익명 인증
- 선택적 Google 로그인 연결
- 공유 코드 또는 초대 링크로 여행 참여
- 공동 편집 사용자인 `TripMember`와 비용 정산 대상인 `Participant`를 분리해 관리
- 네이버 지도 API 기반 장소 검색, 네이버 장소 링크 붙여넣기, 직접 입력으로 장소 저장
- 날짜별 시간표 작성
- 일정 순서에 따른 지도 번호 핀과 직선 동선 표시
- KRW 원 단위 실제 지출 입력
- 지출별 결제자, 소비자, 분할 방식, 참여자별 배분 금액 기록
- 전체 균등, 메뉴·항목별 소비자 지정, 공용 메뉴 균등, 참여자별 직접 부담액 입력 지원
- 할인, 봉사료, 기타 조정 금액 배분
- 개인 결제액, 개인 부담액, 받을 금액 또는 보낼 금액 실시간 재계산
- 카테고리별 개인 소비 합계와 날짜·장소·메뉴/지출별 개인 소비 내역 제공
- 정산 공유 문구 복사
- 영수증 이미지에서 수정 가능한 OCR 항목 초안 생성
- `.trip.json` 내보내기 및 가져오기

MVP에서 사용자는 별도 로그인 화면 없이 바로 여행을 만들 수 있다. 내부적으로는 Firebase Anonymous Auth로 `uid`를 발급받아 사용자를 구분하고, 사용자가 원하면 Google 로그인을 연결해 내 여행 목록과 프로필을 보관한다.

정산에서는 다음 세 값을 구분한다. 개인 결제액은 자신이 `payer`인 지출의 합계이고, 개인 부담액은 `allocatedAmounts`에 배분된 자신의 몫이며, 정산 결과는 `개인 결제액 - 개인 부담액`이다. 결과가 양수면 받을 금액, 음수면 보낼 금액으로 표시한다.

정산 원장의 canonical `Expense`는 `expenseDate`, `payer { participantId, amount }`, `consumers`, `allocationMethod: equal | itemized | custom`, `allocatedAmounts [{ participantId, amount }]`, `receiptItems`를 기준으로 한다.

영수증 OCR은 MVP에 포함하되, 사용자가 항목과 배분을 검토하고 명시적으로 확정하기 전에는 정산 원장에 반영하지 않는다. 사용자가 로컬에서 이미지를 선택하면 Cloud Functions와 CLOVA OCR을 통해 수정 가능한 항목 초안을 만들고, 항목명과 금액 수정, 누락 항목 수동 추가, 항목별 소비자와 배분 방식 지정을 거쳐 저장한다. 인식에 실패하면 전체 금액 기반 수동 등록으로 전환한다. 사용자가 최종 저장하기 전에는 지출과 정산 결과에 반영하지 않으며, 영수증 이미지는 Firebase Storage에 영구 저장하지 않는다.

## 5. 핵심 사용자 경험

사용자는 여행을 만들고 공유 링크나 코드로 공동 편집 멤버를 초대하며, 로그인하지 않은 동행도 정산 참여자로 추가할 수 있다. 가고 싶은 장소를 검색하거나 네이버 장소 링크를 붙여넣어 장소 보관함에 저장하고, 저장한 장소를 날짜별 시간표에 배치한다. 여행 중 또는 여행 후에 실제 지출을 입력하면 앱이 결제자와 소비자별 배분 금액을 기준으로 정산 결과를 실시간으로 갱신한다.

개인 정산 화면에서는 내가 먼저 결제한 총액, 이번 여행에서 실제로 소비한 부담 총액, 받을 금액 또는 보낼 금액을 나란히 확인한다. 이어서 카테고리별 개인 소비 합계와 날짜·장소·메뉴 또는 지출별 상세 내역을 확인해, 송금 관계뿐 아니라 이번 여행에서 무엇에 얼마를 소비했는지도 이해할 수 있어야 한다.

제품의 핵심 흐름은 다음 문장으로 정리할 수 있다.

> 장소를 고르면 일정이 되고, 일정에서 쓴 돈이 자동으로 정산된다.

## 6. 공유 및 동기화 방식

초기 MVP는 Firebase 백엔드를 사용해 여행 정보를 저장하고 공유한다. 여행장은 여행을 생성한 뒤 공유 코드 또는 초대 링크를 전달한다. 사용자는 코드나 링크로 같은 여행 세션의 `TripMember`가 되고, Firestore 실시간 구독을 통해 일정, 장소, 지출 변경을 즉시 확인한다.

기본 공유 흐름은 다음과 같다.

- 앱 접속
- 내부 익명 `uid` 발급
- 여행 생성
- 공유 코드 또는 초대 링크 생성
- 사용자가 코드 입력 또는 링크 접속
- `trips/{tripId}/members/{uid}`에 멤버 등록
- 장소, 일정, 지출 데이터 실시간 구독
- 지출 변경 시 정산 결과 재계산

MVP에서는 공유 코드나 초대 링크로 입장한 모든 `TripMember`가 편집 가능한 단순 권한 모델로 시작한다. 보기 전용, 정산만 보기, 관리자 권한, 변경 이력은 이후 확장한다.

`.trip.json`은 더 이상 핵심 공유 수단이 아니라 백업, 데모 데이터, 데이터 이전, 장애 대비용 보조 기능으로 둔다.

## 7. 국내 우선, 해외 확장

처음에는 국내 여행만 제대로 만든다. 국내 여행에서는 네이버 지도, 한국 주소, 원화 정산, 한국어 영수증 인식을 우선 고려한다.

이후 해외 여행으로 확장할 때는 Google Maps 기반 장소 검색, 외화 및 환율, 국가별 타임존, 다국어 OCR을 추가한다.

MVP의 여행 생성 화면은 국내 여행만 제공한다. 데이터 모델과 provider 경계는 후속 확장을 위해 다음 구분을 수용할 수 있게 설계한다.

- 국내 여행: 네이버 지도, KRW, 한국어 OCR 중심
- 해외 여행(후속): Google Maps, 외화/환율, 다국어 OCR 중심

앱 내부 구조는 국내/해외를 완전히 분리하지 않고, 지도와 OCR 같은 외부 연동만 provider 방식으로 교체할 수 있게 설계한다.

## 8. 차별점

- 일정과 정산을 따로 관리하지 않는다.
- 장소, 일정, 비용이 하나의 데이터로 연결된다.
- Firebase 기반 실시간 여행 세션으로 여러 명이 동시에 편집할 수 있다.
- 익명 인증으로 진입 장벽을 낮추고, Google 로그인으로 내 여행 목록과 프로필을 보관할 수 있다.
- 결제액, 부담액, 정산 결과를 분리해 대납 관계와 개인 소비를 동시에 설명한다.
- 영수증 전체 균등 분할부터 메뉴별 소비자 지정과 직접 부담액 입력까지 실제 식사·숙소 정산 흐름을 지원한다.
- 공동 여행에서 자주 발생하는 일부 참여, 대리 결제, 운전자 보상 같은 케이스로 확장하기 좋다.
- 국내 여행에 맞는 지도와 정산 경험을 먼저 깊게 만든다.
- 네이버 저장 리스트 import에 의존하지 않고 장소 링크와 검색만으로도 일정을 만들 수 있다.
- `.trip.json` 백업으로 Firebase 데이터 이전과 데모 시나리오를 지원한다.

## 9. 이후 확장 아이디어

- 내 여행 목록의 검색·보관·최근 본 여행 기능
- 활동 피드: 누가 장소, 일정, 지출을 수정했는지 표시
- 여행장, 편집자, 보기 전용 권한
- 정산만 보기 공유 링크
- 정산 완료 체크 및 송금 완료 상태
- 정산 결과 snapshot과 확정 이력
- 영수증 문맥을 이용한 소비자 자동 추론
- 이동 시간 기반 일정 추천
- 실제 도로 경로와 교통수단별 예상 시간 계산
- 장소 후보 투표
- 여행 템플릿 저장
- 환율 자동 반영
- 해외 여행 모드에서 Google Maps 연동
- 네이버 저장 목록 자동 가져오기 실험
- 고도화된 공동 편집 및 변경 이력 관리

## 10. 후속 검증 항목

- 공유 코드 만료, 재생성, 최대 사용 횟수를 어디까지 넣을지
- 네이버 장소 링크 붙여넣기에서 장소명, 주소, 좌표를 안정적으로 추출할 수 있는지
- 네이버 저장 목록 자동 가져오기 실험이 공식 API와 약관 범위 안에서 가능한지
- OCR 이미지의 일시 전송·처리·폐기 정책을 사용자에게 어떻게 안내할지
- 매우 긴 영수증에서 지출 문서 크기와 항목 편집 성능을 어떻게 제한할지
- 균등 분할의 1원 나머지와 할인 배분 결과를 모든 클라이언트에서 동일하게 재현하는지
- 이 프로젝트를 포트폴리오 중심으로 볼지, 실제 서비스 가능성까지 볼지

## 11. 1차 목표

기존 강릉 여행 HTML을 바탕으로, 여러 여행에 재사용 가능한 Firebase 기반 모바일 우선 웹/PWA MVP를 만든다. 첫 번째 목표는 네이버 지도와 KRW를 사용하는 완성도 높은 국내 여행 일정, 직선 지도 동선, 개인 소비가 드러나는 실시간 정산 경험을 만드는 것이다.

## 12. 3인 구현 소유권

- 플랫폼·통합: Vite/Firebase 기반, 인증, 여행 생성·공유·멤버, 앱 셸·PWA, 공통 UI, 보안 규칙, 백업, 최종 통합
- 정산·영수증: `Participant`, `Expense`, `ReceiptItem`, 분할·정산 엔진, 개인 소비 화면, 지출 UI, `parseReceipt`와 OCR 검토 흐름
- 장소·일정·지도: 네이버 장소 검색·링크 파싱·직접 입력, 장소 보관함, 일정 편집, 지도 어댑터, 번호 핀과 직선 동선
